## 环境要求

| 项目 | 要求 |
|---|---|
| 操作系统 | Windows（本项目仅支持 Windows） |

## 从源码启动

### 1. 克隆仓库

```bash
git clone https://github.com/anonymousException/renpy-translator.git
cd renpy-translator
```

### 2. 创建虚拟环境（推荐）

```bash
python -m venv venv
venv\Scripts\activate
```

### 3. 安装依赖

```bash
pip install -r src\requirements.txt
```

依赖列表（`src/requirements.txt`）：

- PySide6==6.6.3（Qt6 GUI 框架）
- pygtrans==1.5.3（Google 翻译）
- deepl==1.18.0（DeepL 翻译）
- openai==1.43.0（OpenAI 翻译）
- translators==5.9.2（多引擎翻译）
- pyperclip==1.9.0（剪贴板）
- openpyxl==3.1.5（Excel 读写）
- qt-material==2.14（Material 主题）
- beautifulsoup4==4.12.3（HTML 解析）
- pywin32==306（Windows API）
- ping3==4.0.8（网络检测）
- pyinstaller==6.10.0（PyInstaller 打包，仅编译时需要）
- pyinstaller-hooks-contrib==2024.8（PyInstaller 钩子）

### 4. 启动应用

```bash
python src\main.py
```

应用会启动 Qt6 GUI 窗口，加载 qt-material 主题。

## 使用 PyInstaller 编译

PyInstaller 将 Python 应用打包为独立可执行文件。优点是产物文件较小，缺点是容易被杀毒软件误报。

### 手动步骤

#### 1. 确保依赖已安装

```bash
pip install -r src\requirements.txt
```

#### 2. 执行 PyInstaller 构建

```bash
cd src
pyinstaller main.spec
```

构建完成后，产物位于 `src\dist\main\` 目录。

#### 3. 复制资源文件

PyInstaller 不会自动打包以下资源文件，需要手动复制到产物目录：

```bash
cd ..
xcopy /E src\supported_language src\dist\main\supported_language\
xcopy /E src\resource src\dist\main\resource\
xcopy /E src\qm src\dist\main\qm\
xcopy /E src\custom_engine src\dist\main\custom_engine\
copy src\custom.txt src\dist\main\custom.txt
copy src\hook_unrpa.rpy src\dist\main\hook_unrpa.rpy
copy src\openai_model.txt src\dist\main\openai_model.txt
copy src\openai_template.json src\dist\main\openai_template.json
copy src\hook_extract.rpy src\dist\main\hook_extract.rpy
copy src\hook_add_change_language_entrance.rpy src\dist\main\hook_add_change_language_entrance.rpy
copy src\default_langauge_template.txt src\dist\main\default_langauge_template.txt
copy src\rpatool src\dist\main\rpatool
copy src\font_style_template.txt src\dist\main\font_style_template.txt
copy src\main.ico src\dist\main\main.ico
copy src\cacert.pem src\dist\main\cacert.pem
```

#### 4. 运行

双击 `src\dist\main\main.exe` 即可启动。

## 使用 Nuitka 编译

Nuitka 将 Python 编译为 C 代码再编译为机器码，产物不容易被杀毒软件误报，但产物体积较大且编译时间较长。

### 前置条件

- 安装 Nuitka：`pip install nuitka`
- 需要 C 编译器（如 MSVC 或 MinGW64），Nuitka 首次运行时会提示下载

### 手动步骤

#### 1. 确保依赖已安装

```bash
pip install -r src\requirements.txt
pip install nuitka
```

#### 2. 执行 Nuitka 构建

```bash
python -m nuitka ^
    --standalone ^
    --disable-console ^
    --enable-plugin=pyside6 ^
    --windows-icon-from-ico=src\main.ico ^
    --noinclude-data-files=src\resource ^
    --output-dir=build ^
    src\main.py
```

参数说明：

| 参数 | 说明 |
|---|---|
| `--standalone` | 独立模式，包含所有依赖 |
| `--disable-console` | 不显示控制台窗口 |
| `--enable-plugin=pyside6` | 启用 PySide6 插件支持 |
| `--windows-icon-from-ico` | 设置 exe 图标 |
| `--noinclude-data-files` | 排除 resource 目录（后续手动复制） |
| `--output-dir` | 输出目录 |

构建完成后，产物位于 `build\main.dist\` 目录。

#### 3. 组装发布目录

```bash
mkdir dist\windows
xcopy /E build\main.dist\ dist\windows\
xcopy /E src\resource\ dist\windows\resource\
xcopy src\supported_language dist\windows\supported_language\
xcopy src\qm dist\windows\qm\
xcopy src\custom_engine dist\windows\custom_engine\
copy src\custom.txt dist\windows\custom.txt
copy src\hook_unrpa.rpy dist\windows\hook_unrpa.rpy
copy src\openai_model.txt dist\windows\openai_model.txt
copy src\openai_template.json dist\windows\openai_template.json
copy src\hook_extract.rpy dist\windows\hook_extract.rpy
copy src\hook_add_change_language_entrance.rpy dist\windows\hook_add_change_language_entrance.rpy
copy src\default_langauge_template.txt dist\windows\default_langauge_template.txt
copy src\rpatool dist\windows\rpatool
copy src\font_style_template.txt dist\windows\font_style_template.txt
copy src\cacert.pem dist\windows\cacert.pem
copy src\main.ico dist\windows\main.ico
```

#### 4. 运行

双击 `dist\windows\main.exe` 即可启动。

## 编译脚本一键构建

项目根目录提供了两个一键编译脚本，省去手动操作：

| 脚本 | 说明 |
|---|---|
| `build_pyinstaller.bat` | 使用 PyInstaller 编译，产物在 `src\dist\main\` |
| `build_nuitka.bat` | 使用 Nuitka 编译，产物在 `dist\windows\` |

### 使用方法

```bash
# PyInstaller 编译
build_pyinstaller.bat

# Nuitka 编译
build_nuitka.bat
```

脚本会自动完成：安装依赖 → 编译 → 复制资源文件 → 输出产物路径。

## 常见问题

### Q: 杀毒软件报毒怎么办？

PyInstaller 打包的产物容易被误报为病毒，这是误报（false positive）。建议：
- 使用 Nuitka 编译（`build_nuitka.bat`），大多数杀毒软件不会误报
- 或将产物目录加入杀毒软件白名单

### Q: 编译时报 PySide6 相关错误？

确保 PySide6 版本为 6.6.3：
```bash
pip install pyside6==6.6.3
```

### Q: Nuitka 编译找不到 C 编译器？

Nuitka 需要 C 编译器。推荐安装 Visual Studio Build Tools（包含 MSVC）。
首次运行 Nuitka 时会自动提示下载 MinGW64 作为替代。

### Q: 启动后界面显示异常？

检查 `src/resource/` 目录是否已正确复制到产物目录。该目录包含图标等界面资源。

### Q: 编译后运行闪退？

检查以下资源文件是否已复制到产物目录：
- `supported_language/` — 语言支持文件
- `qm/` — Qt 翻译文件
- `custom_engine/` — 自定义翻译引擎
- `cacert.pem` — SSL 证书
- 各种 `.rpy`、`.txt`、`.json` 配置文件
