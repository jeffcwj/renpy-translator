## 1. 前置准备

### 1.1 游戏解包

一键翻译会自动处理解包，无需手动操作。如果游戏 `game/` 目录下有 `.rpa` 文件，工具会自动解包。

### 1.2 获取 API Key

你需要一个 OpenAI 兼容的 API 服务。常见选择：

| 服务商 | API 地址 | 说明 |
|--------|----------|------|
| OpenAI 官方 | `https://api.openai.com/v1` | 需科学上网，费用较高 |
| 阿里云通义千问 | `https://dashscope.aliyuncs.com/compatible-mode/v1` | 国内直连，性价比高 |
| DeepSeek | `https://api.deepseek.com/v1` | 国内直连 |
| 其他兼容服务 | 各自的 base_url | 只要兼容 OpenAI chat API 即可 |

### 1.3 准备字体文件

翻译为中文/日文/韩文等需要 CJK 字体。推荐：

- **中文**：[思源黑体 (SourceHanSansCN)](https://github.com/CyanoHao/WFM-Free-Font/tree/master/SourceHanSansCN) — Regular 或 Medium 权重
- **日文**：[花園明朝 (hanazomefont)](https://www.asterism-m.com/font/hanazomefont/)
- **注意**：字体文件越大，游戏启动和文字渲染越慢（见第 6 节）

## 2. 引擎配置

1. 打开 Renpy Translator
2. 点击菜单 **引擎设置** (Engine Settings)
3. 选择翻译引擎为 **OpenAI**
4. 填写以下参数：

| 参数 | 说明 |
|------|------|
| **API Key** | 你的 API 密钥 |
| **Base URL** | API 地址（如 `https://dashscope.aliyuncs.com/compatible-mode/v1`） |
| **Model** | 模型名称（如 `qwen-plus`、`gpt-3.5-turbo`） |
| **RPM** | 每分钟最大请求数 |
| **RPS** | 每秒最大请求数 |
| **TPM** | 每分钟最大 token 数 |
| **Timeout** | 请求超时时间（秒） |
| **Max Length** | 每批翻译的最大字符数 |

---

## 3. 参数推荐

### ⚠️ 最重要的参数：Max Length

`max_length` 控制每次发送给 AI 的文本批量大小。**这是影响翻译质量最关键的参数。**

- **设置过大**（如 50000）→ AI 输出超过其最大 token 限制 → JSON 被截断 → 翻译大量丢失
- **设置过小**（如 500）→ 每批只有几行 → 请求次数暴增 → 速度慢、费用高
- **推荐值：3000 ~ 5000**（默认 5000 即可）

### 各模型推荐配置

#### 阿里云通义千问 (Qwen)

| 参数 | qwen-plus（推荐） | qwen-turbo（快速） | qwen3.5-plus |
|------|-------------------|-------------------|--------------|
| RPM | 500 | 500 | 500 |
| RPS | 10 | 10 | 10 |
| TPM | 2000000 | 2000000 | 2000000 |
| Timeout | 120 | 60 | 120 |
| Max Length | **5000** | **3000** | **3000** |

> **qwen3.5-plus 注意**：该模型输出 token 上限较低，`max_length` 务必设为 3000 或更低，否则大批量文本容易被截断。

#### OpenAI 官方

| 参数 | gpt-3.5-turbo | gpt-4 |
|------|--------------|-------|
| RPM | 250 | 100 |
| RPS | 10 | 3 |
| TPM | 根据账号等级 | 根据账号等级 |
| Timeout | 120 | 180 |
| Max Length | **5000** | **5000** |

#### DeepSeek

| 参数 | deepseek-chat |
|------|--------------|
| RPM | 300 |
| RPS | 10 |
| TPM | 2000000 |
| Timeout | 120 |
| Max Length | **5000** |

### 参数调优原则

1. **首次翻译用默认参数或上表推荐值**，观察 `log.txt` 中是否有错误
2. 如果频繁出现 `finish_reason=length` 警告 → **降低 max_length**
3. 如果频繁出现 429 / RateLimitError → **降低 RPM 和 RPS**
4. 如果翻译速度太慢 → 适当提高 RPM/RPS（不超过服务商限额）
5. **不要为了速度盲目增大 max_length**，这是翻译失败的最常见原因

## 4. 首次翻译流程

### 4.1 一键翻译步骤

1. 打开 Renpy Translator
2. 选择 **一键翻译** (One Key Translate) 标签页
3. 填写/选择以下信息：
   - **游戏目录**：选择游戏根目录（包含 `game/` 文件夹的那个目录，通常有 `.exe` 文件）
   - **TL Name**：翻译语言标签（如 `chinese`、`schinese`、`japanese`）
   - **目标语言**：下拉选择目标语言
   - **字体文件**：选择你准备的 CJK 字体文件（`.ttf` 或 `.otf`）
4. 确认引擎配置正确（见第 2、3 节）
5. 点击 **开始翻译**
6. 等待翻译完成，观察日志输出

### 4.2 翻译过程中的日志

翻译过程中，`log.txt` 会记录进度。正常的日志应该频繁出现：

```
part translation success,still in progress,please waiting...
```

如果出现以下警告，说明需要调整参数：

```
WARNING: AI output was truncated (finish_reason=length)...
```

→ 降低 `max_length` 参数（见第 3 节）

### 4.3 翻译完成后

1. 打开游戏，进入 **设置 (Preferences)** 页面
2. 找到 **Language** 选项，切换到你翻译的语言
3. 如果看到乱码 → 字体替换可能失败，重新执行一键翻译中的字体替换步骤
4. 如果部分文本仍为英文 → 这些行翻译时出错被跳过，可以重新翻译（已翻译的行会自动跳过）

## 5. 游戏更新后的翻译迁移

当游戏发布新版本后，你之前的翻译可以部分复用。

### 5.1 迁移步骤

1. **下载新版游戏**，解压到新目录
2. 从旧版游戏中 **复制** 以下内容到新版：
   - `game/tl/{你的tl_name}/` 整个文件夹
   - `game/fonts/` 文件夹（如果有）
3. 用 Renpy Translator 打开新版游戏目录
4. **重新执行一键翻译**（或手动：先提取 → 再翻译）
5. 工具会自动跳过已翻译的行，只翻译新增内容

### 5.2 注意事项

- 如果新版游戏修改了已有对话的原文，对应的旧翻译会被保留但可能不再匹配
- 新版游戏可能新增了 UI 样式或字体设置，建议重新执行字体替换
- 如果迁移后游戏报错，可以使用 **错误修复** 功能（需要游戏 exe 文件）

### 5.3 迁移失败的常见原因

- 新版游戏 **重构了脚本文件结构**（文件改名/拆分） → 旧 tl 文件无法匹配，需要重新翻译
- 新版游戏 **更换了 Ren'Py 引擎版本** → 可能需要重新提取和翻译
- `.rpyc` 缓存冲突 → 删除 `game/tl/{tl_name}/` 下的所有 `.rpyc` 文件后重试

## 6. 翻译后游戏卡顿问题

翻译后切换到翻译语言时，点击按钮或推进对话可能出现明显延迟（约 0.x 秒），而英文原版流畅。

### 原因分析

1. **CJK 字体文件过大**：思源黑体 Regular 约 10-20MB，而英文字体通常只有 100-300KB。Ren'Py 每次渲染文字时都需要在大字体文件中查找字形
2. **翻译文件数量**：`game/tl/` 下生成大量 `.rpy` 文件，Ren'Py 启动时需要编译
3. **Ren'Py 文字渲染机制**：CJK 字符渲染本身比拉丁字母更耗资源

### 缓解措施

1. **使用较小的字体文件**：
   - 思源黑体有多个子集版本，选择 SC (Simplified Chinese) 而非 全语言版
   - 或使用其他较小的中文字体（如文泉驿微米黑 ~5MB）
2. **首次启动让 Ren'Py 编译**：
   - 翻译完成后第一次启动游戏会较慢（编译 .rpy → .rpyc）
   - 之后启动会使用缓存的 .rpyc，速度会改善
3. **关闭不需要的游戏特效**：减轻整体渲染压力

## 7. 翻译后文件夹过大的清理

一键翻译过程中会解包 `.rpa` 档案，释放大量图片、音频、视频等资源文件到 `game/` 目录下。这会导致游戏文件夹显著增大。

### 哪些文件可以清理？

翻译完成后，实际需要的文件只有：

- `game/tl/{tl_name}/` — 翻译文件
- `game/fonts/` — 字体文件
- `game/gui.rpy` 和 `game/screens.rpy` — 可能被修改的配置

解包释放的素材文件（`.webp`、`.webm`、`.png`、`.ogg` 等）如果 `.rpa` 档案仍在，可以安全删除。

### 清理方法

1. **确认 `.rpa` 档案完整**：检查 `game/` 下是否仍有 `.rpa` 文件（如 `archive.rpa`、`images.rpa` 等）
2. **删除解包的素材文件夹**：删除 `game/` 下非必要的图片/音频/视频文件夹
3. **保留** `game/tl/`、`game/fonts/`、`game/*.rpy`、`game/*.rpyc` 和所有 `.rpa` 文件

> ⚠️ 建议在清理前先备份整个游戏文件夹，以防误删

## 8. 自定义翻译提示词模板

翻译质量很大程度上取决于发送给 AI 的提示词 (prompt)。你可以自定义模板来优化翻译效果。

### 模板文件位置

- `src/openai_template.json` — 当前使用的模板
- `src/openai_template.default.json` — 原始默认模板（备份）

### 模板格式

模板是一个 JSON 数组，包含 OpenAI Chat API 的 messages 结构。支持以下占位符：

| 占位符 | 说明 |
|--------|------|
| `#SOURCE_LANGUAGE_ID!@$^#` | 会被替换为源语言名称 |
| `#TARGET_LANGAUGE_ID!@$^#` | 会被替换为目标语言名称 |
| `#JSON_DATA_WAITING_FOR_TRANSLATE_ID!@$^#` | 会被替换为待翻译的 JSON 数据 |

> 注意：`TARGET_LANGAUGE_ID` 中的 "LANGAUGE" 是故意的拼写，不要修改。

### 当前默认模板（Galgame 优化版）

当前默认模板已针对 Galgame/视觉小说翻译优化，特点：

- 保留人名/地名的英文原文
- 口语化、角色感情色彩的翻译风格
- 保护代码控制符（`[name]`、`{player}`、`\n` 等）
- 二次元本地化语气

如需恢复原始默认模板，将 `openai_template.default.json` 复制为 `openai_template.json` 即可。

## 9. 常见错误与解决方案

### 9.1 JSONDecodeError / 截断错误

```
json.decoder.JSONDecodeError: Unterminated string starting at...
```

**原因**：AI 输出被截断，返回了不完整的 JSON。通常是 `max_length` 设置过大导致发送的文本超过模型的最大输出 token 限制。

**解决**：
1. 降低 `max_length` 到 3000-5000
2. 工具已自动尝试修复截断的 JSON并恢复已翻译的部分
3. 未能恢复的会自动拆分成更小的批次重试

### 9.2 translated result can not match

```
translated result can not match the untranslated contents
```

**原因**：AI 返回的翻译结果数量与原文不匹配（可能合并了多行或遗漏了某些行）。

**解决**：
1. 工具会自动将批次拆半重试
2. 如果持续失败，尝试降低 `max_length`
3. 也可以换用其他翻译引擎（如 Google）翻译失败的行

### 9.3 429 RateLimitError

```
A 429 status code was received; we should back off a bit.
```

**原因**：请求频率超过服务商限额。

**解决**：
1. 降低 RPM 和 RPS 参数
2. 检查服务商的账户限额（新账户通常限制很严格）
3. 等待一段时间后重新翻译（已翻译的会自动跳过）

### 9.4 特殊符号翻译错误

```
Error in line:1320
It's [s_name]. And [y_name].
It's [0] . And [1] .
É [0] . E 1] .
```

**原因**：翻译引擎破坏了 `[]`、`{}`、`<>` 等特殊符号的结构。工具会将 `[name]` 替换为 `[0]` 保护，但翻译引擎有时会破坏这个格式。

**解决**：
1. 这种情况很少见，通常只影响极少数行
2. 可以用 Google 翻译引擎重新翻译这些行（Google 对特殊符号更友好）
3. 剩余极少数错误行需要手动修改

### 9.5 连接错误

```
The server could not be reached
```

**解决**：
1. 检查网络连接
2. 检查 Base URL 是否正确
3. 如需科学上网，在工具中配置代理（如 `http://127.0.0.1:7890`）
4. 重新翻译即可（已翻译行自动跳过）

### 9.6 游戏启动后报错

翻译后游戏报错通常是因为特殊符号被破坏（见 9.4）。

**解决**：
1. 使用工具的 **错误修复** 功能（需要指定游戏 .exe 文件）
2. 错误修复会调用 Ren'Py 的 lint 功能自动检测并修复翻译错误
3. 如果自动修复仍无法解决，需要手动打开报错的 .rpy 文件修改对应行
